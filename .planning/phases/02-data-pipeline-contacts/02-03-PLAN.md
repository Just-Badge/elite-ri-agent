**src/trigger/meeting-dispatcher.ts** -- Scheduled cron dispatcher:

Use `schedules.task` from `@trigger.dev/sdk` (NOT the deprecated `client.defineJob`).

```typescript
import { schedules } from "@trigger.dev/sdk";
import { createAdminClient } from "@/lib/supabase/admin";
import { processUserMeetings } from "./process-user-meetings";
```

`meetingDispatcher = schedules.task({
  id: "meeting-processing-dispatcher",
  cron: "0 */1 * * *",  // Every hour
  run: async () => { ... }
})`

The run function:
1. Use admin client to query all user_settings where granola_refresh_token_encrypted IS NOT NULL (has Granola connected)
2. For each user, parse their `processing_schedule` JSONB: `{ interval_hours, start_hour, end_hour, timezone }`
3. Check if current hour in the user's timezone falls within their start_hour..end_hour window
4. If within window, trigger `processUserMeetings` with `{ userId: user.user_id }`
5. Use per-user queue with concurrency limit 1: `queue: { name: \`user-\${user.user_id}-meetings\`, concurrencyLimit: 1 }`

Helper function to get current hour in timezone: Use `Intl.DateTimeFormat` with `timeZone` option and `hour` format to get the hour number.

**src/trigger/process-user-meetings.ts** -- Per-user processing task:

```typescript
import { task, logger } from "@trigger.dev/sdk";
```

`processUserMeetings = task({
  id: "process-user-meetings",
  retry: { maxAttempts: 3, factor: 1.8, minTimeoutInMs: 1000, maxTimeoutInMs: 30000 },
  run: async (payload: { userId: string }) => { ... }
})`

The run function:
1. Get access token: `getOrRefreshAccessToken(payload.userId)`
2. Get user's z.ai API key from user_settings (decrypt `zai_api_key_encrypted`)
3. If no API key, log warning and return early (cannot do AI extraction)
4. Fetch recent Granola documents: `getGranolaDocuments(accessToken, 50)` (last 50 docs)
5. Get list of already-processed granola_document_ids from meetings table for this user
6. For each new (unprocessed) document:
   a. Fetch transcript: `getGranolaTranscript(accessToken, doc.id)`
   b. Combine transcript segments into a single text string
   c. Insert meeting record into meetings table (granola_document_id, title, meeting_date, granola_url=`https://app.granola.so/notes/${doc.id}`, transcript)
   d. Get existing contacts for this user (name, email) for dedup context
   e. Call `extractContactsFromTranscript(apiKey, fullTranscript, doc.title, existingContacts)`
   f. For each extracted contact:
      - If contact has email AND email matches existing contact: UPDATE the existing contact (merge new info)
      - If contact has email AND no match: INSERT new contact
      - If contact has no email but name matches: UPDATE existing (merge cautiously)
      - If contact has no email and no match: INSERT with ai_confidence from extraction
      - Create contact_meetings junction record
      - Insert action_items linked to both contact and meeting
   g. Add 200ms delay between transcript fetches (rate limit protection)
7. Return `{ documentsProcessed, contactsCreated, contactsUpdated }`

Use `createAdminClient()` for ALL database operations (Trigger.dev tasks run outside user session).
Always filter by user_id explicitly since admin client bypasses RLS.

**IMPORTANT for Task 3 testability:** Extract the contact dedup/upsert logic (step 6f) into a named, exported helper function `upsertExtractedContacts` in process-user-meetings.ts. This function takes `(supabase, userId, meetingId, extractedContacts, existingContacts)` and returns `{ created: number, updated: number }`. This enables unit testing of dedup logic independently from Granola API calls.

Similarly, extract meeting record insertion (step 6c) into a helper `insertMeetingRecord(supabase, userId, doc)` that returns the inserted meeting row including `granola_url`. This enables unit testing of URL format and junction creation.

POST handler:
1. Auth check (same pattern)
2. Parse body expecting `{ refresh_token: string, client_id: string }`
