```sql
CREATE TABLE contacts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  company TEXT,
  role TEXT,
  location TEXT,

  background TEXT,
  relationship_context JSONB DEFAULT '{}',
  status TEXT DEFAULT 'active' CHECK (status IN ('active', 'not_pursuing', 'dormant')),
  outreach_frequency_days INTEGER DEFAULT 30,
  last_interaction_at TIMESTAMPTZ,
  notes TEXT,
  ai_confidence TEXT CHECK (ai_confidence IN ('high', 'medium', 'low', 'manual')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, email)
);
```

**src/lib/validations/contacts.ts** -- Create Zod schemas following the pattern from settings.ts:

```typescript
import { z } from "zod";

const CONTACT_CATEGORIES = [
  "accelerators", "advisors", "clients", "investors",
  "networking", "partners", "potential-team", "team",
] as const;

export const contactSchema = z.object({
  name: z.string().min(1, "Name is required").max(200),
  email: z.string().email().optional().or(z.literal("")),
  company: z.string().max(200).optional(),
  role: z.string().max(200).optional(),
  location: z.string().max(200).optional(),
  connected_via: z.string().max(500).optional(),
  category: z.enum(CONTACT_CATEGORIES).optional(),
  background: z.string().max(5000).optional(),
  relationship_context: z.object({
    why: z.string().optional(),
    what: z.string().optional(),
    mutual_value: z.string().optional(),
  }).optional(),
  status: z.enum(["active", "not_pursuing", "dormant"]).default("active"),
  outreach_frequency_days: z.number().int().min(1).max(365).default(30),
  notes: z.string().max(10000).optional(),
});

export const contactUpdateSchema = contactSchema.partial().extend({
  id: z.string().uuid(),
});

export type ContactFormValues = z.infer<typeof contactSchema>;
export type ContactUpdateValues = z.infer<typeof contactUpdateSchema>;
export { CONTACT_CATEGORIES };
```

**src/lib/seed/md-parser.ts** -- Parse seed MD files into structured data:

Export `ParsedContact` interface and `parseSeedContactMd(content: string, category: string)` function.

The parser must handle these structural variations found in the 289 seed files:
- Bold contact info: `- **Email:** x` AND plain: `- Email: x`
- Relationship context as subsections (`### Why We Connected`) AND as inline paragraphs
- Action items with checkboxes `- [ ] task` AND plain `- task`
- Meeting history with `### date -- title` subsections with `[Granola Notes](url)` links
- Status values: "Active", "Not Pursuing", "Nurturing" (map "Nurturing" to "active")
- Missing sections should return undefined/empty, not throw

Use regex-based section extraction (NOT a full markdown AST parser -- per research recommendations).

Extract sections by splitting on `## ` headings, then parse each section independently.

**src/lib/seed/import-contacts.ts** -- Import function:

Export `importSeedContacts(userId: string, relationshipsDir: string)` that:
1. Reads all `.md` files from all category subdirectories using fs
2. Parses each with `parseSeedContactMd`
3. Inserts contacts using admin client (bypasses RLS) with the provided userId
4. For each contact's meetings: insert into meetings table, create contact_meetings junction
5. For each contact's action items: insert into action_items table linked to contact
6. Returns `{ contactsImported: number, meetingsImported: number, actionItemsImported: number }`

Use `createAdminClient()` from `@/lib/supabase/admin`. Filter by user_id explicitly (admin client bypasses RLS).
