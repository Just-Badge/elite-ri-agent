**src/lib/granola/client.ts** -- HTTP client with token rotation:

Constants:
- `GRANOLA_BASE = "https://api.granola.ai"`
- `WORKOS_AUTH = "https://api.workos.com/user_management/authenticate"`
- `TOKEN_BUFFER_MS = 5 * 60 * 1000` (5 minutes before expiry, refresh proactively)

Export these functions:

`refreshGranolaToken(userId: string, currentRefreshToken: string, clientId: string): Promise<GranolaTokens>`:
- POST to WORKOS_AUTH with `{ client_id: clientId, grant_type: "refresh_token", refresh_token: currentRefreshToken }`
- On success, IMMEDIATELY persist the new refresh_token encrypted to user_settings via admin client (this is CRITICAL -- the old token is now invalid)
- Update granola_token_expiry and granola_token_status='active' in same update
- Return GranolaTokens with access_token, refresh_token, expires_at
- On non-OK response, throw descriptive error

`getOrRefreshAccessToken(userId: string): Promise<string>`:
- Read user_settings for this userId via admin client: granola_refresh_token_encrypted, granola_client_id, granola_token_expiry
- If token expiry is in the future and more than TOKEN_BUFFER_MS away, decrypt and return cached access token (note: we do NOT cache access tokens in user_settings -- this function should maintain an in-memory cache or always refresh; for v1, always refresh since tokens last 1 hour and tasks run infrequently)
- Actually for v1: ALWAYS call refreshGranolaToken since we only store refresh tokens, not access tokens. Each task run gets a fresh access token.
- Decrypt the stored refresh token, call refreshGranolaToken, return the new access_token

`getGranolaDocuments(accessToken: string, limit?: number, offset?: number): Promise<GranolaDocumentsResponse>`:
- POST to `${GRANOLA_BASE}/v2/get-documents` with Bearer accessToken
- Body: `{ limit: limit || 100, offset: offset || 0, include_last_viewed_panel: true }`
- Throw on non-OK response

`getGranolaTranscript(accessToken: string, documentId: string): Promise<GranolaTranscriptSegment[]>`:
- POST to `${GRANOLA_BASE}/v1/get-document-transcript` with Bearer accessToken
- Body: `{ document_id: documentId }`
- Throw on non-OK response

**CRITICAL ANTI-PATTERN**: Never reuse a consumed refresh token. Each call to refreshGranolaToken invalidates the previous token. The new token MUST be persisted before any other work.

**src/lib/ai/types.ts** -- Types and Zod schema for AI extraction:

```typescript
import { z } from "zod";

export const extractedContactSchema = z.object({
  name: z.string(),
  email: z.string().email().optional().nullable(),
  company: z.string().optional().nullable(),
  role: z.string().optional().nullable(),
  location: z.string().optional().nullable(),
  category: z.enum([
    "accelerators", "advisors", "clients", "investors",
    "networking", "partners", "potential-team", "team"
  ]).optional().nullable(),
  background: z.string().optional().nullable(),
  relationship_context: z.object({
    why: z.string().optional().nullable(),
    what: z.string().optional().nullable(),
    mutual_value: z.string().optional().nullable(),
  }).optional().nullable(),
  action_items: z.array(z.string()).optional().nullable(),
  notes: z.string().optional().nullable(),
  confidence: z.enum(["high", "medium", "low"]),
});

export const contactExtractionSchema = z.object({
  contacts: z.array(extractedContactSchema),
  meeting_summary: z.string(),
});

export type ExtractedContact = z.infer<typeof extractedContactSchema>;
export type ExtractionResult = z.infer<typeof contactExtractionSchema>;
```

**src/lib/ai/extract-contacts.ts** -- AI extraction function:

Export `extractContactsFromTranscript(apiKey: string, transcript: string, meetingTitle: string, existingContacts: { name: string; email?: string }[]): Promise<ExtractionResult>`

Implementation:
1. Create OpenAI client: `new OpenAI({ apiKey, baseURL: "https://api.z.ai/api/paas/v4" })`
2. System prompt instructs the AI to:
   - Extract contact info for each person mentioned (other than the meeting owner)
   - For each person: name, email (if mentioned), company, role, location, category suggestion, background summary, relationship context (why connected, what discussed, mutual value), action items, brief notes
   - Assign confidence: "high" if name AND email are clear, "medium" if name clear but email uncertain/missing, "low" if identity is ambiguous
   - Reference EXISTING CONTACTS list for deduplication (match by email if possible, by name as secondary)
   - Also produce a meeting_summary (2-3 sentence overview)
   - Return valid JSON matching the schema
3. User message: `Meeting: "${meetingTitle}"\n\nTranscript:\n${transcript}`
4. Call `client.chat.completions.create({ model: "glm-5", messages, response_format: { type: "json_object" }, temperature: 0.3, max_tokens: 4096 })`
5. Parse response content with `contactExtractionSchema.parse(JSON.parse(content))`
6. If response.choices[0]?.message?.content is null/empty, throw `new Error("No AI response content")`
7. Return parsed result
