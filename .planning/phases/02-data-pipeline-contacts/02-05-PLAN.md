```typescript
interface ContactDetailProps {
  contactId: string;
}
```

State: contact (fetched data), loading, saving, editMode.

On mount: fetch `GET /api/contacts/${contactId}` and set contact.

Layout using shadcn Card sections:
1. **Header section**: Contact name (large), category Badge, status indicator, "Edit" / "Cancel" toggle button, "Delete" button (with confirmation)
2. **Contact Info section** (read-only when not editing): email, company, role, location, connected_via
3. **Edit form** (visible when editMode is true): `<ContactForm contact={contact} onSave={handleSave} saving={saving} />`
   - handleSave: PUT `/api/contacts/${contactId}` with form values, toast on success, refetch contact, exit edit mode
4. **Relationship Context section** (read-only when not editing): Why We Connected, What Was Discussed, Mutual Value
5. **Background section**: background text
6. **Meeting History section**: `<MeetingHistory meetings={contact.contact_meetings?.map(cm => cm.meetings) || []} />`
7. **Action Items section**: `<ActionItems items={contact.action_items || []} onToggle={handleToggleAction} />`
   - handleToggleAction: PUT `/api/contacts/${contactId}/action-items` with `{ actionItemId, completed }`
8. **Notes section**: notes text
9. **Outreach section**: outreach_frequency_days display, last_interaction_at display

Delete handler: confirm dialog, then DELETE `/api/contacts/${contactId}`, redirect to /contacts on success.

Back link at top: "Back to Contacts" link to /contacts.
