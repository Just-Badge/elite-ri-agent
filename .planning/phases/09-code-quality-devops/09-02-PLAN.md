Pattern:
```typescript
function requireEnv(name: string): string {
  const value = process.env[name];
  if (!value) throw new Error(`Missing required environment variable: ${name}`);
  return value;
}

// Validates on first import -- fail fast
export const env = {
  NEXT_PUBLIC_SUPABASE_URL: requireEnv("NEXT_PUBLIC_SUPABASE_URL"),
  NEXT_PUBLIC_SUPABASE_ANON_KEY: requireEnv("NEXT_PUBLIC_SUPABASE_ANON_KEY"),
  SUPABASE_SERVICE_ROLE_KEY: requireEnv("SUPABASE_SERVICE_ROLE_KEY"),
  ENCRYPTION_KEY: requireEnv("ENCRYPTION_KEY"),
  GOOGLE_CLIENT_ID: requireEnv("GOOGLE_CLIENT_ID"),
  GOOGLE_CLIENT_SECRET: requireEnv("GOOGLE_CLIENT_SECRET"),
} as const;
```

However, collect ALL missing vars before throwing so the developer sees every missing var at once:

```typescript
const required = [
  "NEXT_PUBLIC_SUPABASE_URL",
  "NEXT_PUBLIC_SUPABASE_ANON_KEY",
  "SUPABASE_SERVICE_ROLE_KEY",
  "ENCRYPTION_KEY",
  "GOOGLE_CLIENT_ID",
  "GOOGLE_CLIENT_SECRET",
] as const;

const missing = required.filter((name) => !process.env[name]);
if (missing.length > 0) {
  throw new Error(
    `Missing required environment variables:\n${missing.map((n) => `  - ${n}`).join("\n")}`
  );
}
```

Then export typed `env` object with each var as a string property.

Write the test file using dynamic import to test both success and failure cases. In the test, manipulate `process.env` before each dynamic import (use `vi.resetModules()` to clear the import cache between tests).

DO NOT wire this module into the existing supabase/gmail/crypto files yet -- that's out of scope. This plan only creates the module; wiring happens if needed in a future plan.
