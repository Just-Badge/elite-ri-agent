**layout.tsx -- Mobile header overflow prevention:**

1. Add `overflow-hidden` to the header element and `min-w-0` to the Breadcrumbs container to prevent long breadcrumb text from causing horizontal overflow:
   ```
   <header className="flex h-14 shrink-0 items-center gap-2 border-b px-4 overflow-hidden">
     <SidebarTrigger />
     <Separator orientation="vertical" className="mr-2 h-4" />
     <div className="min-w-0 flex-1">
       <Breadcrumbs />
     </div>
     <div className="ml-auto shrink-0">
       <ThemeToggle />
     </div>
   </header>
   ```
   The `min-w-0` on the breadcrumb wrapper allows it to shrink below its content size, and `shrink-0` on ThemeToggle prevents it from being crushed.

2. On the `<main>` element, reduce padding on mobile:
   ```
   <main id="main-content" tabIndex={-1} className="flex-1 p-4 sm:p-6 outline-none">
   ```
   This gives more space on small screens.

**draft-card.tsx -- Button wrapping on mobile:**

3. Change the action buttons container from `flex items-center gap-2` to `flex flex-wrap items-center gap-2` so buttons wrap to a second line on narrow screens instead of overflowing:
   ```
   <div className="flex flex-wrap items-center gap-2 pt-2">
   ```

**contacts/page.tsx -- Responsive header:**

4. The page header has `flex items-start justify-between` for the title and "Process Meetings" button. On very narrow screens the button may overlap. Wrap this in a responsive pattern:
   ```
   <div className="flex flex-col gap-3 sm:flex-row sm:items-start sm:justify-between">
   ```
   This stacks the heading and button on mobile, side-by-side on larger screens.

5. The filter section already uses `flex flex-col gap-4 sm:flex-row` -- this is correct. Ensure the Select has `w-full sm:w-[180px]` instead of just `w-[180px]` so it takes full width on mobile:
   ```
   <SelectTrigger className="w-full sm:w-[180px]">
   ```

**drafts/page.tsx -- Responsive header:**

6. Same pattern for the drafts header -- make it stack on mobile:

   ```

   ```
